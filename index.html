Simple GitHub CMD Guide

Learn GitHub step-by-step through essential commands, explained in a simple and beginner-friendly manner.

Git and GitHub Basics

Before diving into commands, let's understand some basic concepts about Git and GitHub.

What is Git?

Git is a version control system that tracks changes to files and allows multiple people to work on the same project without conflicting with each other. It helps developers manage their code effectively.

What is GitHub?

GitHub is a platform for hosting and managing Git repositories. It allows developers to collaborate on projects by providing features like pull requests, issues, and actions.

Why Use GitHub?

GitHub is widely used for both open-source and private projects. You can work on personal projects or collaborate with other developers worldwide.

How to Upload a React Project to GitHub

Here's how to upload a React project to GitHub in simple steps:

1. Open your terminal or command prompt:

Open Command Prompt/Terminal on your computer.

2. Go to your project folder:

cd path/to/your/react-project
This command moves you inside your React project's folder. Replace path/to/your/react-project with the actual location.

3. Initialize git (if not already done):

git init
This command starts tracking your project using Git (Git is like the brain of your project).

4. Add your project files to git:

git add .
This tells Git to prepare all files for uploading.

5. Save your changes:

git commit -m "My first commit"
This step saves your progress with a message describing your changes (like writing in a diary).

6. Link your local project to GitHub:

git remote add origin https://github.com/yourusername/repository-name.git
Replace the URL with your GitHub repository link (you can get this after creating a repo on GitHub).

7. Push (upload) the code to GitHub:

git push -u origin main
Finally, this command uploads your project to GitHub.

Example:

If your React project folder is on your desktop, here’s how you would upload it:

cd Desktop/react-project
git init
git add .
git commit -m "Initial React project commit"
git remote add origin https://github.com/username/my-react-project.git
git push -u origin main

How to Upload Any Project to GitHub

You can upload any kind of project to GitHub using similar steps. Here's the simplified version:

1. Open terminal and go to your project folder:

cd path/to/your/project-folder

2. Initialize git:

git init

3. Add your files:

git add .

4. Commit your changes:

git commit -m "Added all files"

5. Link your GitHub repo:

git remote add origin https://github.com/username/project.git

6. Push to GitHub:

git push -u origin main

Example:

If your project folder is on your D: drive:

cd D:/projects/my-awesome-project
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/awesome-project.git
git push -u origin main

Collaborating on GitHub

GitHub makes it easy to work with others. Here’s how you can collaborate on a project with GitHub.

Forking a Repository

Forking a repository means copying someone else's project to your GitHub account so you can work on it. You can then submit a pull request to merge your changes back into the original project.

git clone https://github.com/username/repository-name.git
This command copies the repository from GitHub to your local machine.

Creating and Switching Branches

When collaborating, it's good practice to create a new branch for each feature or bug fix.

git checkout -b new-feature-branch
This command creates and switches to a new branch named new-feature-branch.

Common Git Commands Cheats

  • Check the status: git status - See what changes have been made in your project.
  • View commit history: git log - View all saved changes.
  • Switch branches: git checkout branch-name - Move to another version of your project.
  • Pull updates: git pull - Download the latest changes from GitHub.
  • Clone a repo: git clone https://github.com/username/repo-name.git - Copy an entire project from GitHub to your computer.

Common Git Issues and How to Fix Them

Sometimes, you might run into problems while using Git. Here are some common issues and how to fix them.

1. Git Push Rejected

This error occurs when the branch you're trying to push is behind the branch on GitHub.

Error: Updates were rejected because the remote contains work that you do not have locally.

Fix: You need to pull the changes from the remote branch first:

git pull origin main

2. Permission Denied (SSH)

This error occurs when your SSH keys are not set up correctly for GitHub.

Error: Permission denied (publickey).

Fix: You need to set up your SSH keys:

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
